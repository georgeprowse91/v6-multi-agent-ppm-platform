# Release Plan

## Purpose
Define scope, timing, dependencies, and readiness criteria for a release so stakeholders can
align on delivery outcomes and operational readiness.

## Audience
Product Owner, Release Manager, Engineering Lead, QA Lead, Operations, Stakeholders.

## Inputs
- Sprint plans (`docs/methodology/agile/templates/sprint-plan.md`).
- Backlog priorities (`docs/methodology/agile/templates/backlog.csv`).
- Risk and change logs.

## Definitions
- **Release window:** The target timeframe for deployment and customer rollout.
- **Readiness checklist:** Required criteria for a safe and compliant release.

## Methodology Gate Alignment
- Agile gate: Sprint Review Gate (`docs/methodology/agile/gates.yaml`).
- Hybrid gate: Release Readiness Gate (`docs/methodology/hybrid/gates.yaml`).

## Required Sections
- Release overview
- Scope summary
- Feature commitments
- Dependencies and assumptions
- Quality & risk management
- Release readiness checklist
- Communications plan
- Approvals
