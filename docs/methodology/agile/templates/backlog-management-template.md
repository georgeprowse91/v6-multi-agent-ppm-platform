# Backlog Management Template

## Purpose
Provide a structured view of backlog health, prioritization, and readiness to support
release planning and sprint execution.

## When to Use
- Use for ongoing backlog governance, prioritization, and health reviews.
- Use to summarize readiness for sprint planning and release commitments.
- Use alongside (not instead of) backlog refinement notes for session-level detail.

## Audience
Product Owner, Scrum Master, delivery team, and portfolio stakeholders.

## Inputs
- Product strategy and roadmap themes.
- Customer feedback and usage analytics.
- Delivery capacity and dependency maps.

## 3. Prioritization Framework
### Prioritization Criteria
| Criteria | Weight | Description |
| --- | --- | --- |
| Business Value | 30% | Revenue impact, cost savings, strategic alignment |
| User Impact | 25% | User satisfaction, experience improvement |
| Technical Risk | 20% | Implementation complexity, technical debt |
| Dependencies | 15% | Blocking other work, external dependencies |
| Effort Required | 10% | Development time, resource requirements |

### Priority Levels
- **P0 - Critical:** Must have, business critical
- **P1 - High:** Should have, significant value
- **P2 - Medium:** Could have, moderate value
- **P3 - Low:** Nice to have, minimal value

## 6. Acceptance Criteria Standards
### Story Requirements
- [ ] Clear user story format (As a... I want... So that...)
- [ ] Specific, measurable acceptance criteria
- [ ] Given-When-Then scenarios where appropriate
- [ ] UI/UX requirements defined
- [ ] Performance criteria specified
- [ ] Security considerations noted

### Definition of Ready
- [ ] Story is clearly understood by the team
- [ ] Acceptance criteria are defined
- [ ] Story is estimated
- [ ] Dependencies are identified
- [ ] Designs/mockups available (if needed)
- [ ] Technical approach agreed upon

### Definition of Done
- [ ] All acceptance criteria met
- [ ] Code reviewed and approved
- [ ] Tests written and passing
- [ ] Documentation updated
- [ ] Product Owner acceptance
- [ ] No known defects
